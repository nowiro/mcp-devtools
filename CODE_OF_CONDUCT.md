# Code of Conduct — mcp-devtools

Ten projekt przyjmuje [Contributor Covenant v2.1](https://www.contributor-covenant.org/version/2/1/code_of_conduct/) jako swój kodeks postępowania.

## Skrót

W ramach kontrybucji do tego repozytorium — przez issues, PRs, code review, komentarze, discussions, e-maile do maintainerów — zobowiązujemy się do:

- **Szacunku.** Każdy uczestnik zasługuje na uprzejmość, niezależnie od doświadczenia, narodowości, identyfikacji, religii czy poglądów.
- **Konstruktywności.** Krytyka dotyczy kodu i decyzji, nie osób. Sugerujemy alternatywy zamiast tylko negować.
- **Cierpliwości.** Pamiętamy, że nie każdy ma ten sam kontekst — pytania początkujących są mile widziane.
- **Profesjonalizmu.** Bez harassmentu, doxxingu, atakowania osób spoza technicznej merytoryki.

Niedopuszczalne są w szczególności:

- Język nacechowany seksualnie lub obraźliwy.
- Trolling, obraźliwe komentarze, ataki personalne lub polityczne.
- Publikowanie prywatnych informacji innych osób (adres, e-mail, …) bez ich wyraźnej zgody.
- Każde inne zachowanie, które rozsądnie można uznać za nieprofesjonalne w środowisku zawodowym.

## Egzekwowanie

Naruszenia można zgłaszać do maintainerów przez **prywatne kanały** (GitHub private vulnerability report dla incydentów security; e-mail maintainerów dla pozostałych — patrz [SUPPORT.md](SUPPORT.md) po adresy).

Maintainerzy zobowiązują się:

- Potwierdzić otrzymanie zgłoszenia w ciągu 7 dni roboczych.
- Przeprowadzić obiektywne dochodzenie.
- Zachować poufność co do tożsamości zgłaszającego, na ile to praktyczne.
- Podjąć proporcjonalne działania (od ostrzeżenia, przez tymczasowe zawieszenie udziału, po permanentny ban i zgłoszenie do GitHub Trust & Safety w przypadkach skrajnych).
